	Face Rig/Mesh-based metarig system
		(inspired by what I saw on BlenRig's page)
		- User spawns a pre-made mesh.
		- Under the hood there's an armature that's copying vertex positions on this mesh
		- Shape this mesh to their own character
		- Press a button that "generates" a metarig from the bones, by just applying the pose, removing some constraints, and revealing the armature.
		- Have pre-made Actions
		- Operator to merge Armatures without losing Action Slots.
		- This system can be used for body rigs too, but those are already pretty easy to adjust anyways.
		- Bone rolls and eyeballs might pose minor issues

	Constraint relinking could be a whole lot simpler if we could use a PREFIX in the constraint name to define which bone the constraint should be moved to.
		Eg. "STR-TIP=Armature@DEF-Head@DEF-Jaw" would move this constraint to the bone prefixed "STR-TIP", with the rest of the name same as this bone, and put DEF-Head and DEF-Jaw as the targets.
			No more weird crazy relinking rules
				although the weird rules can still be used as default, I would rather get rid of them and do backwards comp with metarig versioning.
